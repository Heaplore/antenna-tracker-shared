## 1. Problem Statement

**谁在什么场景下遇到了什么问题？**

技术质量管理人员在日常工作中需要掌握天线行业的动态信息，但目前面临三大痛点：

1. **信息碎片化**：原材料价格、企业竞争格局、行业标准、技术趋势分散在不同渠道，需要手动搜集整理
2. **缺乏分析能力**：即使有了数据，也难以从多维度（技术/质量/成本/交付）进行交叉分析，无法形成决策支持
3. **知识无法复用**：每天的技术解读笔记、行业分析停留在个人层面，没有结构化的知识沉淀和关联

**核心价值主张**：
> 天线情报追踪系统帮助技术质量管理人员**一站式获取行业情报、交叉分析多维数据、沉淀行业知识网络**，从"被动收集信息"升级为"主动决策支持"。

**Evidence**:
- 老大日常技术/质量/成本/交付管理工作中，天线行业信息是核心数据源之一
- 小月每天输出技术解读笔记，但缺乏结构化的知识管理和关联
- 现有框架已定义 6 大板块 + 分析决策层 + 知识图谱，方向明确

## 3. Non-Goals

- **不构建实时交互的大模型对话界面**（静态网站限制，已在框架设计中否决）
- **不做移动端 App**（第一阶段聚焦 Web 端）
- **不替代专业金融终端**（如 Wind、Bloomberg），只做天线行业垂直领域的情报展示
- **不做用户注册/登录系统**（内部工具，非 SaaS 产品）

## 4. User Personas & Stories

**Primary Persona**: 技术质量管理人员（老大本人）
- 日常工作涉及技术选型、质量控制、成本核算、交付管理
- 需要快速掌握天线行业最新动态
- 关注维度：技术趋势、质量标准、原材料价格、竞争对手动向

**Core User Stories**:

**Story 1**: 作为技术质量管理人员，我希望一站式查看所有板块数据，这样我不需要切换多个网站/文档。
- [ ] 给定首页，当用户访问时，显示 6 大板块概览 + KPI 指标
- [ ] 给定导航栏，当用户点击板块时，跳转到对应页面

**Story 2**: 作为技术质量管理人员，我希望知识图谱能串联各板块数据，这样我能发现板块间的关联关系。
- [ ] 给定知识图谱页面，当用户点击节点时，显示详情侧边栏
- [ ] 给定关系边，当用户悬停时，高亮关联节点

**Story 3**: 作为技术质量管理人员，我希望能看到四维交叉分析结论，这样我能做出更明智的决策。
- [ ] 给定分析决策层，当 LLM 处理多板块数据时，输出交叉分析结论
- [ ] 给定建议卡片，当分析完成时，显示量化建议和风险提示

## 5. Solution Overview

### MVP 范围（Phase 1-3）

**Phase 1：市场板块试点**（2 周）
- 完成"抓取 → 清洗 → 入库 → 验证"完整 pipeline
- 前端展示原材料价格趋势（铜、铝、PCB、PTFE）
- 知识图谱接入小月技术解读笔记

**Phase 2：知识图谱 + 分析决策层**（2 周）
- 知识图谱可视化（力导向图 + 双层 Wiki）
- LLM 四维交叉分析引擎
- 建议卡片展示

**Phase 3：其他板块扩展**（4 周）
- 企业、标准、行业动态、技术板块逐个接入
- 知识图谱逐步完善
- 全局搜索功能

### Key Design Decisions:
- **技术选型**：Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui + recharts（已验证，无需变更）
- **数据形态**：JSON 静态文件（零部署成本，适合静态导出）
- **部署方式**：Gitee Pages（免费，符合老大"零成本"原则）
- **LLM 集成**：分析决策层使用 MiniMax-M3（已有 API，成本低）

## 6. Technical Considerations

**Dependencies**:
- **飞书日历 API** — 用于日程同步（可选，非核心）
- **MiniMax-M3 API** — 用于 LLM 分析决策层（核心依赖）
- **长江有色金属网/卓创资讯** — 原材料价格数据源（核心依赖）
- **全国标准信息公共服务平台** — 标准数据源（核心依赖）

**Known Risks**:
| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| 数据源 API 变更或封禁 | Medium | High | 多数据源冗余 + 手动录入 fallback |
| Gitee Pages 部署限制 | Low | Medium | 备选 GitHub Pages |
| LLM 分析结果不准确 | Medium | Medium | 人工审核 + 置信度标注 |
