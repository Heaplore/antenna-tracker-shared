有源天线单元 AAU深度解读 Active Antenna Unit

AAU = 把 4G 时代的 BBU + RRU + 无源天线三件套，塞进一个盒子，挂在铁塔上直接干活。天线不再是"哑巴喇叭"，而是能"思考"的智能辐射体。

一、从 4G 到 5G：基站架构的三代演变

第一代：BBU + 无源天线（2G/3G 时代）

基带处理单元（BBU）放在机房，信号通过馈线直接连到无源天线。天线就是一个金属反射面，信号从馈线进来，往一个方向撒出去。天线本身没有任何"脑子"——它不控制相位、不调波束，全凭发射端的功率来决定覆盖。

缺点：馈线损耗大（尤其高频段），覆盖方向固定，无法针对用户优化。

第二代：BBU + RRU + 无源天线（4G LTE 时代）

RRU（射频拉远单元）被放到塔上，靠近天线，用光纤连接 BBU。RRU 负责数模转换、功放放大，但天线本身仍然是"哑巴"——它只管辐射，不管波束控制。

优点：缩短了馈线长度，降低了损耗。

缺点：天线还是不能"思考"，波束倾角靠机械调节（拧螺丝），无法动态变化。

第三代：BBU + DU/CU + AAU（5G NR 时代）

AAU 把 RRU 的射频功能 + 波束赋形芯片 + 大规模天线阵列，全部集成到一个户外单元里。天线不再是被动辐射体，而是能实时调整波束方向的"智能辐射体"。

架构变化：

  • BBU 被拆分为 DU（分布式单元）和 CU（集中式单元）
  • DU 负责实时调度和物理层处理，靠近 AAU 部署
  • CU 负责非实时功能和核心网对接，集中在机房
4G:  BBU(机房) ---光纤--- RRU(塔上) ---馈线--- 无源天线(塔上)
    5G:  CU(机房) ---光纤--- DU(机房/塔下) ---光纤--- AAU(塔上，集成天线)

关键变化：馈线消失了，取而代之的是光纤连接 AAU。因为 5G 高频段（3.5GHz）的馈线损耗太大——100 米馈线在 3.5GHz 的损耗超过 10dB，相当于信号衰减了 90%。光纤几乎没有损耗，所以 AAU 必须把射频功能做到天线旁边。

二、AAU 的内部解剖：一个盒子里装了什么？

AAU 不是简单的"RRU + 天线"拼装货，它是一个高度集成的系统工程。核心组件包括：

1. 射频收发芯片（Transceiver）

每个射频通道配备一颗收发芯片，负责数模转换（DAC/ADC）、上变频/下变频。64T64R 的 AAU 内部有 128 颗收发芯片（64 发 + 64 收）。

类比：每颗芯片就是一个"翻译官"，把数字信号翻译成无线电波发出去，再把收到的无线电波翻译回数字信号。

2. 功率放大器（PA）

这是 AAU 最耗电的部件，占总功耗的 50%-70%。64T64R 的 AAU 满载功耗 1000-1400W，其中功放就吃了大半。

材料演进：

  • 4G 时代：硅基 LDMOS 功放，效率约 30-40%
  • 5G 初期：GaAs（砷化镓）功放，效率提升到 45-50%
  • 2025-2026 主流：GaN（氮化镓）功放，效率达到 55-60%，同时功率密度更高、耐高温

GaN 的优势：氮化镓的禁带宽度比硅大 3 倍，能在更高电压、更高频率下工作而不击穿。这意味着同样的输出功率，GaN 功放体积更小、发热更少、效率更高。

3. 移相器与衰减器

每根天线通道配备一个移相器和一个衰减器，通过控制信号的相位和幅度，实现波束赋形。64 通道 = 64 个移相器 + 64 个衰减器。

移相器的精度：典型精度为 5.625°（2^5 bit 相位控制），精度越高，波束指向越准，但成本和功耗也越高。

4. 天线阵列（Radiating Elements）

这是 AAU 的"面子"。64T64R 的 AAU 通常包含 192-384 个辐射单元（每个通道由多个单元组成阵列，以提升增益）。

排列方式：

  • 一维排列：早期 4T4R/8T8R 天线，只在水平方向形成波束
  • 二维排列：5G Massive MIMO 的标准配置，在水平和垂直方向同时形成波束，实现 3D 波束赋形

尺寸：一副典型的 64T64R AAU 尺寸约为 250mm × 300mm × 100mm，重量 15-25kg，挂在铁塔上像一个白色的长方形面板。

5. 散热系统

64 通道功放堆在一起，满载时发热量巨大。AAU 内部采用：

  • 铝合金外壳：兼作散热器，表面积大
  • 导热凝胶：填充芯片与外壳之间的空隙，降低热阻
  • 无风扇设计：户外部署不能用风扇（灰尘、雨水），靠自然对流和辐射散热

温度控制：AAU 内部设有多个温度传感器，当温度超过阈值时，会自动降低发射功率（thermal throttling），防止过热损坏。夏季铁塔表面温度可达 60°C 以上，AAU 的散热设计必须在最恶劣环境下仍能保证芯片结温不超过 105°C。

6. 基带处理模块

部分 AAU 内置轻量级基带处理能力，负责波束赋形算法的实时运算。但更常见的做法是把基带处理放在 DU 中，AAU 只负责射频和天线控制。

三、为什么 5G 非要用 AAU？三个底层原因

原因 1：高频段衰减太狠，必须"打激光"

5G 主力频段在 3.5GHz（n78）和 4.9GHz，比 4G 的 2.1GHz 频率更高。频率越高，自由空间路径损耗越大：

自由空间损耗公式: L = 32.4 + 20log₁₀(d) + 20log₁₀(f)
    (d 单位 km, f 单位 MHz)
  • 2.1GHz（4G）1km 损耗：约 100dB
  • 3.5GHz（5G）1km 损耗：约 103dB
  • 4.9GHz（5G）1km 损耗：约 105dB

3.5GHz 比 2.1GHz 多损耗 3dB，意味着信号强度减半。靠传统"全向辐射"根本覆盖不了。AAU 通过大规模天线阵列形成高增益定向波束，用 64 根天线的阵列增益（约 18dB）把信号"打"出去——相当于用放大镜聚光，能量集中了，距离自然就远了。

原因 2：用户密度爆发，需要"多车道并行"

一个体育场里几万人同时刷视频、发朋友圈，4G 那种"一人一份、排队服务"的模式扛不住。AAU 的多用户 MIMO（MU-MIMO）能力，可以在同一频段、同一时间，对不同用户形成独立的波束。

类比：4G 基站像一个餐厅服务员，一次只能给一桌客人端菜；AAU 像一个有 64 只手的服务员，同时给 64 桌客人上菜，每道菜都不串味。

实测数据：64T64R Massive MIMO 相比 4T4R，小区容量提升 3-5 倍，边缘用户速率提升 10 倍以上。

原因 3：能耗压力，必须"精打细算"

5G 基站的功耗是 4G 的 2-3 倍。一副 64T64R AAU 满载功耗 1000-1400W，而 4G 4T4R RRU 满载仅约 300-400W。但 5G 的单比特能耗（Watt/bit）反而比 4G 低了 20 倍——问题是用户量还没涨到那个程度，大量基站处于轻载甚至空载状态，功耗浪费严重。

AAU 提供了多层级的节能机制：

节能模式关断范围功耗唤醒时间
正常模式无1000-1400W0
符号级静默关闭部分 OFDM 符号的功放800-1000W微秒级
通道休眠关闭空闲射频通道（64→32）500-700W毫秒级
载波休眠关闭空闲频段400-600W秒级
深度休眠关闭大部分有源模块<10W分钟级
极致休眠（2026）仅保留电源和 MCU<5W3 分钟内

行业案例：中国联通和华为合作的"极致休眠"方案，在闲时时段 AAU 功耗降至 5W 以下（对比基线节能增益提升 99%），实现"0 Bit 0 Watt"理念。单模块日均节电 2.2 度，单站预计每年可节省约 2400 度电。

中兴的"休眠"方案：通过软件升级即可激活，无需额外硬件。idle 模式下功耗降至 3W（比家用 LED 灯还低），唤醒响应时间约 1 分钟。已在中國 35 万+ 5G AAU 上部署，年均节能 7500 万度电。

四、AAU 的硬件架构演进

传统架构：馈线 + 功分器 + 天线

传统 Massive MIMO 天线内部有一个无源功分网络（CFN，Combined Feeding Network），把 64 路射频信号通过电缆分配到 192 个辐射单元。问题是：电缆越长，损耗越大；功分器越多，插入损耗越高。

SDIF 架构：信号直注馈电（华为 MetaAAU 首创）

SDIF（Signal Direct Injection Feeding）取消了中间的无源功分网络，每个收发通道直接连接到专属的辐射子阵列。好处：

  • 消除 CFN 插入损耗，辐射效率提升
  • 减少单元间互耦，波束纯度更高
  • 简化内部布线，降低制造成本

华为 MetaAAU 采用 SDIF + 384 偶极子阵列（是传统 64T64R 的 2 倍），实现上下行覆盖各提升 3dB，小区半径扩展 18-20%，同时能耗降低约 30%。

UWB 超宽带架构：一机多频

华为 UWB AAU 系列通过宽带介质滤波器和多频段偶极子技术，在同一副天线面板上集成多个频段（如 3.5GHz + 4.9GHz）。相比传统方案：

  • 不增加尺寸和重量就能新增频段
  • 减少铁塔上的设备数量，降低风载和租金
  • 功耗降低约 15%

全球已部署超过 150 个运营商网络。秘鲁案例：TDD 双频 AAU 部署后覆盖提升 30%，平均功耗降低 30%。

五、PanoAAU：2026 年的新物种

华为在 2026 年推出了 PanoAAU（全景天线），这是 AAU 的又一次进化。核心创新：

从 120° 扇区到 180° 广角覆盖。传统基站每站分 3 个扇区（每个 120°），PanoAAU 通过超大规模天线阵列 + 广角波束管理算法，一个单元就能覆盖 180°。这意味着：

  • 原来 3 个扇区的站点，现在 2 个就够了
  • 铁塔上的设备减少 33%
  • 功耗和租金同步下降
  • 在农村等 ROI 偏低的场景，让原本不划算的站点变得可行

类比：传统基站像一个三个扇面的折扇，PanoAAU 把它展开成半圆——少一把扇子，照样遮住同样的面积。

早期部署案例：赞比亚 MTN 与华为合作的双频段 AAU 部署，解决了塔上空间有限、4G/5G 共站、能耗高等问题。

七、工程里最让人头秃的三个坑

坑 1：散热

64 通道功放堆在一起，满载时发热量惊人。AAU 的散热设计面临三重挑战：

  • 环境温度高：夏季铁塔表面温度可达 60-70°C
  • 密闭空间：户外防水设计（IP65）意味着散热不能靠通风
  • 功率密度大：1400W 塞在 0.02 立方米的空间里，功率密度超过 60kW/m³

解决方案：铝合金外壳 + 导热凝胶 + 内部散热鳍片。但即便如此，高温天高峰期仍可能出现 thermal throttling（过热降频），导致用户感知速率下降。

坑 2：供电

AAU 的功耗是 4G RRU 的 3 倍，这对站点供电系统提出了全新挑战：

  • 传统 4G 站点：平均功耗 6kW，电池后备 8 小时
  • 5G C-band 站点：平均功耗 11.5-14kW，峰值可达 19kW
  • 后备电池_runtime：同样容量的电池，runtime 从 8 小时降到 3-4 小时

更麻烦的是长距离供电时的电压降。AAU 通常工作在 -48V DC，如果供电电缆超过 50 米，线损导致的电压降可能让 AAU 端电压低于工作下限。华为在杭州萧山站点的实测中就遇到了这个问题，最终采用了智能升压技术。

坑 3：O-RAN 互操作性

O-RAN 的愿景是让不同厂商的 AAU 和 DU 能混用，打破华为/爱立信/诺基亚的封闭生态。但现实是：

  • 各家的 Open fronthaul 接口标准还没完全对齐
  • 64T64R 的波束赋形算法高度依赖芯片和固件的配合
  • 运营商不敢轻易换——毕竟基站是网络的命脉

目前主流方案仍然是"封闭盒子"：华为 AAU + 华为 DU，爱立信 AAU + 爱立信 DU。O-RAN 的开放愿景很美，但落地还需要时间。

八、AAU 的未来演进

趋势 1：超大规模阵列（MetaAAU / ELAA）

阵列规模从 192 个偶极子（64T64R）翻倍到 384 个（MetaAAU）。波束更窄、能量更聚焦，覆盖距离更远。华为 MetaAAU 实测数据：上行覆盖 +30%，下行覆盖 +30%，能耗 -30%。

趋势 2：超宽带多频集成（UWB AAU）

一机多频，3.5GHz + 4.9GHz 甚至更低频段集成在同一面板。减少铁塔设备数量，降低风载和租金。全球 150+ 运营商已部署。

趋势 3：广角覆盖（PanoAAU）

从 120° 扇区升级到 180° 覆盖，减少站点和扇区数量，降低 CAPEX 和 OPEX。特别适合农村和偏远地区覆盖。

趋势 4：AI 驱动的节能

基于深度强化学习的智能节能平台，自动识别基站业务的潮汐规律（白天忙、半夜闲），动态调整休眠策略。中国联通的"智能运维机器人"已在试点。

趋势 5：6G 时代的通感一体化 AAU

6G 的 AAU 不仅要通信，还要具备雷达感知能力——同时监测周围环境、追踪移动目标、感知天气变化。一根天线两份能力，赋能自动驾驶、无人机管控、低空经济。

九、给非专业读者的一句话总结

AAU 是 5G 基站的天之骄子，也是工程落地的麻烦制造者。它让天线从"被动辐射体"变成了"智能辐射体"，本质上是通信行业第一次把"计算"搬到了天线边上。

金句：AAU 不是"把信号加大"，而是"把信号送到对的地方"——64 根天线在盒子里对齐步伐，只为让你手机右上角的那一格信号更稳。
