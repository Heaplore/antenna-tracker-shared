AAU 节能技术 AAU

5G AAU 功耗是 4G 的 2.5-3 倍（单站 600-800W vs 200-300W），运营商电费压力巨大。节能技术分三层：符号关断（毫秒级，省电 5-10%）、通道关断（秒级，省电 20-40%）、深度休眠（分钟级，省电 50-70%），配合 AI 预测流量动态调整。

类比入口

AAU 节能就像办公室下班关灯——

  • 符号关断：没人说话时关掉麦克风（毫秒级）
  • 通道关断：人少了关掉一半工位（秒级）
  • 深度休眠：整层楼关灯锁门，有人来时再开（分钟级）

一、5G AAU 功耗构成

1.1 功耗分布

AAU 总功耗：600-800W（64T64R, 3.5GHz）
    
    功耗分布：
      ┌─────────────────────────────────────┐
      │ 功率放大器 PA       │  450-550W  │ 60-70%  │
      │ 数字处理 Board      │   80-120W  │ 15-20%  │
      │ 射频前端（移相器等） │   30-50W   │  5-8%   │
      │ 散热系统            │   20-40W   │  3-5%   │
      │ 其他（电源/控制）    │   30-50W   │  5-8%   │
      └─────────────────────────────────────┘
    
    关键：PA 占 60-70%，是节能主战场

1.2 为什么 5G 更耗电？

1. 天线数多：64T/128T vs 4G 2T/4T → 射频链路多 16-64 倍
    2. 带宽大：100MHz vs 20MHz → 数据处理量大 5 倍
    3. 功放效率低：Doherty PA 平均效率 25-35% → 65-75% 变成热量
    4. 散热需求：AAU 密闭腔体 → 需要风扇/液冷

二、三层节能技术

2.1 符号级关断（Symbol Shutdown）

原理：
      5G OFDM 帧中，空闲时隙没有数据传输
      → 关闭 PA 和射频链路供电
      → 下一个时隙再快速唤醒
      
    时间尺度：
      OFDM 符号周期：~30μs（15kHz 子载波间隔）
      唤醒延迟：< 1μs
      
    省电效果：
      轻载时（< 20% 利用率）：省电 5-10%
      
    实现方式：
      - PA 偏置电压快速调节
      - DAC/ADC 时钟门控
      - 射频开关快速切断
      
    挑战：
      - 唤醒瞬态会产生瞬态干扰
      - 需要精确的定时控制
      - PA 偏置电路响应速度要求高

2.2 通道级关断（Channel/Element Shutdown）

原理：
      用户少时，关闭部分天线通道
      → 从 64T 降到 32T/16T/8T
      → 波束变宽，覆盖范围缩小，但省电
      
    时间尺度：
      秒级到分钟级（基于负载均衡）
      
    省电效果：
      64T → 32T：省电 20-30%
      64T → 16T：省电 40-50%
      
    实现方式：
      - 数字预编码矩阵降秩
      - 关闭部分 RF 链路的 DAC/PA
      - 调整波束宽度补偿覆盖
      
    关键：覆盖补偿
      通道减少后波束变宽 → 边缘用户信号变弱
      → 需要调整天线倾角/功率补偿
      
    3GPP 方案：
      - Deep Sleep Mode：部分通道休眠
      - Dynamic MIMO Power Saving：NR Rel-16 引入

2.3 深度休眠（Deep Sleep）

原理：
      夜间/低峰期，整扇区休眠
      → 只保留最低功耗的监控通道
      → 用户来了快速唤醒
      
    时间尺度：
      分钟级到小时级
      
    省电效果：
      相比常开：省电 50-70%
      
    实现方式：
      - 关闭 PA、大部分射频链路
      - 保留同步信号和唤醒检测
      - 唤醒时间：< 10ms（NR Rel-16 要求）
      
    挑战：
      - 唤醒延迟影响用户体验
      - 频繁开关加速器件老化
      - 需要精准的流量预测

三、AI 节能

3.1 流量预测

基于历史数据的 AI 预测：
      - LSTM/Transformer 预测未来 15min-2h 流量
      - 准确率 > 90%（城市宏站）
      - 提前调整节能策略
      
    应用场景：
      1. 白天高峰：全通道开启，最大化容量
      2. 午间低谷：通道级关断
      3. 深夜：深度休眠 + 符号关断
      4. 突发流量：AI 预测到即将高峰，提前唤醒

3.2 智能波束调整

AI 优化波束指向：
      - 根据用户分布动态调整波束
      - 关闭无人方向的波束
      - 集中能量到有用户的方向
      
    省电效果：
      相比固定波束：额外省电 10-15%

五、一句话总结

5G AAU 节能是"三层防御"：符号关断保毫秒级响应，通道关断应对小时级波动，深度休眠覆盖夜间低谷。AI 预测让节能更精准——不是简单粗暴地"关灯"，而是"知道什么时候该开灯"。记住：节能不是牺牲覆盖，而是在保证 QoS 的前提下，让每一度电都花在刀刃上。
