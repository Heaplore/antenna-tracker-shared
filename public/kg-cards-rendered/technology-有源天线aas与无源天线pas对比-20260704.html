有源天线 vs 无源天线 AAS vs PAS

有源天线（AAS）= 天线 + T/R 模块 + 波束赋形，能主动控制波束方向；无源天线（PAS）= 纯金属辐射体，被动接收/发射信号。5G 时代 AAS 取代 PAS 成为基站标配，因为波束赋形能提升 3-5dB 增益。

类比入口

无源天线就像"手电筒"——光线往哪照是固定的（靠机械转动）。有源天线就像"激光笔"——光束可以电子控制方向，还能同时照多个目标（多波束）。

一、无源天线（PAS - Passive Antenna System）

1.1 基本结构

PAS 架构：
      ┌─────────────────────────────┐
      │   天线辐射体（振子/贴片）     │
      │   反射板/方向图控制          │
      ├─────────────────────────────┤
      │   馈电网络（无源）           │
      ├─────────────────────────────┤
      │   馈线（同轴/波导）          │
      │   （传输信号到天面）         │
      ├─────────────────────────────┤
      │   RRU（射频拉远单元）        │
      │   （在机房/塔底）            │
      └─────────────────────────────┘
      
    信号路径：
      基站(BBU) → 光纤 → RRU → 馈线 → 天线 → 空间
      
    特点：
      - 天线是无源的（不含电子元件）
      - 波束方向靠机械调节（下倾角）
      - 信号在馈线中有损耗（0.5-3dB/100m）
      - 不支持波束赋形

1.2 PAS 优缺点

优点：
      ✓ 结构简单，成本低
      ✓ 技术成熟，维护方便
      ✓ 馈线损耗可控（短距离）
      ✗ 波束方向固定（机械调节慢）
      ✗ 不支持波束赋形
      ✗ 馈线损耗大（尤其毫米波）
      ✗ 多用户只能分时复用

1.3 PAS 典型参数

参数值说明
增益15-18dBi单频点
波束宽度6°-15°水平/垂直
下倾角0-15°机械调节
馈线损耗0.5-3dB/100m取决于频率/长度
极化方式±45°双极化
寿命15-20 年无有源件

二、有源天线（AAS - Active Antenna System）

2.1 基本结构

AAS 架构（5G AAU）：
      ┌─────────────────────────────┐
      │   天线辐射体（64/128/256 单元）│
      ├─────────────────────────────┤
      │   T/R 模块阵列（每通道 1 个）  │
      │   （PA/LNA/移相器/衰减器）   │
      ├─────────────────────────────┤
      │   波束赋形网络（数字/模拟）   │
      ├─────────────────────────────┤
      │   基带接口（CPRI/eCPRI）    │
      └─────────────────────────────┘
      
    信号路径：
      基站(BBU) → 光纤 → AAU（内含 RRu + 天线）
      
    AAU 内部：
      - 数字波束赋形（DBF）：基带处理 → 多通道 DAC → T/R
      - 模拟波束赋形（ABF）：基带处理 → 移相器 → T/R
      - 混合波束赋形（HBF）：数字 + 模拟结合
      
    特点：
      - 天线和有源电路集成
      - 支持 3D 波束赋形
      - 无馈线损耗（天线直连 T/R）
      - 多用户 MIMO

2.2 AAS 优缺点

优点：
      ✓ 波束赋形增益 3-5dB
      ✓ 多用户 MIMO（空间复用）
      ✓ 无馈线损耗
      ✓ 3D 波束赋形（水平+垂直）
      ✓ 节能（动态波束关断）
      ✓ 支持 Massive MIMO（64T64R+）
      ✗ 成本高（T/R 模块贵）
      ✗ 散热挑战大（AAU 功耗 300-800W）
      ✗ 维护复杂（有源件寿命 5-8 年）
      ✗ 需要智能控制算法

2.3 AAS 典型参数

参数值说明
通道数64T64R / 128T128R5G 主流
增益20-25dBi波束赋形后
波束数64-256动态波束
下倾角0-30°电调+数字
功耗300-800W取决于通道数
重量30-60kgAAU
T/R 模块数64-256每通道 1 个

三、PAS → AAS 演进

3.1 代际对比

4G 时代：PAS 为主
      BBU + 光纤 + RRU + 馈线 + 无源天线
      
    5G 初期：AAS 起步
      BBU + 光纤 + AAU（RRU+天线一体化）
      
    5G-A：AAS 增强
      AAU + AI 波束管理 + 通感一体
      
    6G：全有源
      CAPA（连续孔径）+ 全息 MIMO
      天线 = 一面墙（连续辐射面）

3.2 成本对比

项目PAS（4G）AAS（5G）差异
天线单元$50-100$200-500×3-5
T/R 模块无$50-200/通道新增
馈线$1-5/m无节省
RRU$500-1000集成在 AAU合并
AAU 整机不适用$2000-5000新设备
安装成本低高（更重）×1.5
能耗低（100-200W）高（300-800W）×3-4

3.3 为什么 5G 必须用 AAS？

1. 频段升高 → 馈线损耗不可接受
       4G 2.6GHz：馈线损耗 ~0.3dB/100m
       5G 3.5GHz：馈线损耗 ~0.5dB/100m
       5G 28GHz：馈线损耗 ~10dB/100m（毫米波！）
       
    2. Massive MIMO → 需要每通道独立控制
       64T64R = 64 个独立波束
       PAS 做不到
       
    3. 3D 波束赋形 → 需要垂直维度阵列
       PAS 只能机械下倾
       AAS 可以电子下倾 + 水平波束
       
    4. 节能 → 需要动态波束管理
       PAS 全向辐射（浪费能量）
       AAS 按需照射（省电 30-50%）

四、AAS 关键技术

4.1 波束赋形

PAS（无源）：
      固定波束 → 覆盖整个扇区（120°）
      → 所有用户共享同一波束
      → 边缘用户信号弱
      
    AAS（有源）：
      动态波束 → 指向特定用户
      → 每个用户独立波束
      → 边缘用户也能获得高增益
      
    波束数量：
      64T64R：最多 64 个独立波束
      128T128R：最多 128 个独立波束
      
    波束切换时间：
      模拟波束：~1ms
      数字波束：~0.1ms（更快）

4.2 T/R 模块

T/R 模块 = 发射 + 接收一体化模块
      
    发射通道：
      DAC → 衰减器 → 移相器 → PA → 天线
      
    接收通道：
      天线 → LNA → 移相器 → 衰减器 → ADC
      
    关键参数：
      - 移相器分辨率：5/6/7 bit（0.7°-2.8°）
      - 衰减器范围：0-31.5dB（6-bit）
      - PA 输出功率：20-40W（GaN）
      - LNA 噪声系数：1-2dB
      
    成本占比：
      T/R 模块占 AAU 硬件成本 40-50%
      是 AAU 最贵的部件！

五、一句话总结

AAS 是 5G 的"必选项"而非"可选项"——没有波束赋形，Massive MIMO 就是空谈。PAS 到 AAS 的本质变化是"从被动辐射到主动控制"：天线不再是"一根铁疙瘩"，而是一个"智能辐射体阵列"。记住：AAS 虽然性能好，但功耗是 PAS 的 3-4 倍，节能成了 5G 运营商最大的 OPEX 痛点。QMS 要关注的核心指标是"T/R 模块一致性"——64 个通道，只要有一个相位偏差 5°，波束就歪了。
