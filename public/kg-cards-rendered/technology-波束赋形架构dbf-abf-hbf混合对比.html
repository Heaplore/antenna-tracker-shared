波束赋形架构对比 DBF / ABF / HBF

波束赋形三大架构——模拟（ABF）只有一条射频链路、便宜但只能形成一个波束；数字（DBF）每根天线一条链路、最灵活但最贵；混合（HBF）折中，5G 基站主流方案用 64 通道配 12-64 条射频链路，在性能和成本之间找平衡。

类比入口

波束赋形就像"手电筒阵列"——

  • ABF 是 64 个小灯泡共用一个电池，一起亮一起灭，只能控制方向不能控制每个灯的亮度
  • DBF 是 64 个小灯泡各有一个独立开关，可以精确控制每个灯的亮度和开关
  • HBF 是把 64 个灯分成 16 组，每组一个开关，比 ABF 灵活得多，又比 DBF 省钱

一、模拟波束赋形（Analog Beamforming, ABF）

1.1 架构

射频链路（1条）→ 移相器网络 → 天线阵列（N 个单元）
                          │
                      衰减器网络
          
    每个天线单元前有一个移相器（Phase Shifter）
    和/或衰减器（Attenuator）

1.2 特点

特性值/说明
射频链路数1
空间流数1（单用户）
移相器位数2-6 bit（步进 90°-5.6°）
波束切换速度μs 级（模拟域）
功耗极低（1 条 RF 链路）
成本最低
灵活性低（只能形成预置波束）
适用场景毫米波初始接入、卫星通信、4G 传统天线

1.3 核心组件

移相器类型：
      1. 数字移相器（LUT 查找表）
         - 相位步进：360°/2^n（n=位数）
         - 4 bit → 22.5° 步进
         - 6 bit → 5.6° 步进
         
      2. 模拟移相器（PLL/RC 网络）
         - 连续可调
         - 相位精度受温度影响
         
      3. 介质移相器
         - 基于高 Dk 陶瓷
         - 体积小，适合 5G AAU
         
      4. MEMS 移相器
         - 微机电系统
         - 超低损耗，但可靠性待验证

1.4 局限性

1. 只能形成单波束 → 无法多用户复用
    2. 波束倾斜（Beam Squint）→ 宽带信号下不同频率波束指向不同
    3. 移相器相位量化误差 → 波束指向精度受限
    4. 无法自适应零陷 → 干扰抑制能力弱

二、数字波束赋形（Digital Beamforming, DBF）

2.1 架构

射频链路 1 → DAC → 上变频器 → PA → 天线单元 1
    射频链路 2 → DAC → 上变频器 → PA → 天线单元 2
    ...
    射频链路 N → DAC → 上变频器 → PA → 天线单元 N
    
    每根天线都有独立的 RF 链路 → 数字域做波束权重

2.2 特点

特性值/说明
射频链路数N（等于天线数）
空间流数最多 N 流
波束形成数字域预编码矩阵 W ∈ C^(N×K)
波束切换速度ns 级（数字信号处理）
功耗高（N 条 RF 链路）
成本最高
灵活性最高（每根天线独立控制）
适用场景Sub-6GHz 基站、实验室验证

2.3 预编码算法

全数字预编码：
      y = Ws + n
      
      其中：
      y: 接收信号
      W: 预编码矩阵（N_t × N_s）
      s: 发送符号
      n: 噪声
      
    主流算法：
      1. 迫零 ZF（Zero-Forcing）
         W = H^H(HH^H)^(-1)
         简单但噪声增强
         
      2. MMSE（最小均方误差）
         W = H^H(HH^H + σ²I)^(-1)
         考虑噪声
         
      3. WMMSE（加权 MMSE）
         迭代优化，逼近容量
         
      4. SVD 分解
         H = UΣV^H
         最优但计算量大

2.4 在 5G 中的实际应用

64T64R AAU：
      - 64 条射频链路
      - 64 个 DAC/ADC
      - 64 个 PA/LNA
      - 功耗：~500W-600W（含功放）
      
    128T128R AAU：
      - 128 条射频链路
      - 功耗：~800W-1000W
      
    挑战：
      - 功耗随天线数线性增长
      - 成本高昂（每多一条链路 = 多一套 RF 器件）
      - 散热压力大

三、混合波束赋形（Hybrid Beamforming, HBF）

3.1 架构

数字域（基带）：
      K 条射频链路 → K 个 DAC → 模拟预编码矩阵 F_RF
      
    模拟域（射频）：
      F_RF → 移相器网络 → N 根天线
      
    K << N（射频链路数远小于天线数）

3.2 三种连接架构

1. 全连接架构（Fully Connected）
       每条 RF 链路连接所有天线单元
       - 性能好（逼近 DBF）
       - 移相器数量：K × N
       - 功耗高
       - 复杂度最高
       
    2. 部分连接架构（Partially Connected / Subarray）
       每条 RF 链路连接一组天线（子阵）
       - 移相器数量：N
       - 功耗中
       - 5G 基站主流方案
       
    3. 动态子阵架构（Dynamic Subarray）
       射频链路可以动态分配到不同子阵
       - 灵活性最高
       - 需要 RF 开关网络
       - 6G 候选方案

3.3 特点

特性ABFHBF（子阵）DBF
射频链路数1K（12-64）N（64-128）
空间流数1KN
频谱效率基准85-95%100%
功耗基准60-70%100%
成本基准50-60%100%
波束灵活性低中高
5G 主流❌✅实验室

3.4 5G 基站 HBF 典型配置

厂商产品天线数RF 链路数架构
华为AAU 64T64R192（64 端口双极化）64子阵

四、波束倾斜（Beam Squint）问题

问题：
      模拟/混合波束赋形使用相位移相器
      → 相位权重与频率相关
      → 宽带信号下不同频率波束指向不同
      
    公式：
      Δφ = 2πd·sin(θ)/λ = 2πd·f·sin(θ)/c
      
    当 f 变化时，θ 也变化 → 波束倾斜
      
    影响：
      - 宽带系统（> 100MHz）性能下降
      - 毫米波更严重（相对带宽更大）
      
    解决方案：
      1. 真时延（True Time Delay, TTD）
         - 使用时延线代替移相器
         - 所有频率相位一致
         - 成本高、体积大
         
      2. 子带划分
         - 将宽带分成多个子带
         - 每个子带独立波束赋形
         - 折中方案
         
      3. 数字波束赋形
         - 数字域补偿
         - 最彻底但最贵

五、一句话总结

波束赋形架构选择是"性能-功耗-成本"的经典 trade-off：ABF 最简单但只能单用户，DBF 最灵活但功耗成本太高，HBF 折中——5G 基站主流用子阵架构（64T64R 配 32-64 条 RF 链路），频谱效率损失 5-15%，功耗降低 30-40%。记住：6G 时代可能会回到 DBF（GaN/SiGe 工艺进步降低功耗），也可能走向 TTD 混合架构（解决波束倾斜）。
