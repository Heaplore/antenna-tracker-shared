波控板 Beam Control Board

波控板是 AAU 波束管理的"执行中枢"——它接收 DU/gNB 的波束赋形指令，将 64 个通道的幅度和相位权重下发到每个 T/R 模块，实现多波束并发和波束切换。

一、波控板的功能

1.1 核心职责

波控板 = RCU + 波束权重管理 + T/R 模块协调
    
    职责：
      1. 接收波束赋形权重（IQ 数据 + 波束参数）
      2. 计算各通道移相器和衰减器设置
      3. 下发权重到 T/R 模块
      4. 管理多波束并发
      5. 波束切换协调
      6. 校准参数管理

1.2 波束管理流程

1. gNB 调度器决定当前波束配置
    2. 波束权重计算（基带/DU）
    3. 权重通过前传下发到 AAU
    4. 波控板解析权重
    5. 波控板更新 T/R 模块移相器/衰减器
    6. 波束切换完成（< 1ms）

二、波束权重管理

2.1 权重格式

O-RAN O-RU 规范定义的权重格式：
    
    幅度：0-1（12-bit 精度，0.00024 步进）
    相位：0-360°（12-bit 精度，0.088° 步进）
    
    64T64R 示例：
      波束 1：64 个通道的幅度和相位
      波束 2：64 个通道的幅度和相位
      ...
      波束 8：64 个通道的幅度和相位
      
    总数据量：8 × 64 × (12+12) bits = 12.288Kbits

2.2 多波束并发

5G AAU 多波束场景：
      波束 1 → 用户 A（中心区域）
      波束 2 → 用户 B（边缘区域）
      波束 3 → 用户 C（干扰规避方向）
      
    波束隔离度要求：
      相邻波束：> 15dB
      非相邻波束：> 20dB

三、波束切换

3.1 切换机制

波束切换类型：
      1. 快速切换（< 1ms）
         - CSI-RS 测量触发
         - 预计算权重，快速切换
         
      2. 慢速切换（10-100ms）
         - 用户移动触发
         - 重新计算权重
         
      3. 节能切换
         - 夜间减少波束数量
         - 合并覆盖区域

3.2 切换延迟预算

总切换延迟：
      权重下发：100μs
      T/R 模块更新：50μs
      稳定时间：500μs
      总计：~700μs（< 1ms 达标）

四、校准管理

4.1 校准参数存储

波控板存储：
      1. 出厂校准系数（幅度/相位）
      2. 温度补偿查找表
      3. 频率补偿查找表
      4. 最新校准结果

4.2 校准触发

校准触发条件：
      1. 开机时（全通道校准）
      2. 温度变化 > 5°C
      3. 定时校准（每 6 小时）
      4. 手动触发（网管命令）

五、O-RAN 兼容性

5.1 O-RU 接口

O-RAN 规范要求：
      1. 波束权重通过 F1-U 接口下发
      2. 支持 O-RAN Beam Management Profile
      3. 支持动态波束配置
      4. 支持波束切换延迟 < 1ms

5.2 波束管理 O-RAN Profile

Profile 要求：
      - 最大波束数：8
      - 波束切换延迟：< 1ms
      - 幅度精度：12-bit
      - 相位精度：12-bit
      - 波束隔离度：> 15dB

六、一句话总结

波控板是 AAU 波束管理的执行中枢——管理多波束并发、波束切换和校准参数。O-RAN 规范要求波束切换 < 1ms，幅度/相位精度 12-bit。记住：波控板不是简单的"转发器"，它需要协调 64 个 T/R 模块的同步更新，时序管理是关键。
