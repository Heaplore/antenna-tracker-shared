RCU远程控制器单元 Remote Control Unit

RCU 是基站天线的"大脑"——它负责控制天线的所有远程功能：电调下倾角（RET）、波束赋形参数、通道校准、故障诊断和告警上报。在 5G AAU 中，RCU 通常集成在 AAU 本体或 RRU 中，不再作为独立盒子。

类比入口

RCU 就像智能家居的"中控面板"——你可以远程控制窗帘的开合角度（电调下倾）、灯光的亮度（波束赋形）、空调的温度（通道功率），还能知道哪个设备坏了（故障诊断）。

一、定义与功能

1.1 RCU 的角色

传统基站（宏站）：
      BBU（基带）→ 光纤 → RRU（射频）→ 馈线 → 天线
      RCU 是独立控制器，安装在塔下
    
    5G AAU（有源天线）：
      BBU/DU → 前传光纤 → AAU（集成 RCU 功能）
      RCU 功能集成在 AAU 主控板中

1.2 核心功能

功能说明
电调下倾控制远程调节波束下倾角（0-15°）
波束赋形控制调节各通道幅度和相位
通道校准管理 T/R 模块校准流程
功率控制远程调节发射功率
故障诊断监测温度、电压、电流、PIM
告警上报通过 SNMP/NETCONF 上报
软件升级OTA 远程固件升级

1.3 RCU 架构

RCU 硬件组成：
      ├── 主控 MCU（ARM Cortex-M4）
      ├── 通信接口
      │     ├── RS485（连接电调板/波控板）
      │     ├── Ethernet（SNMP/NETCONF）
      │     └── IPMI（基站管理）
      ├── 电源管理（48V DC → 5V/3.3V）
      ├── 保护电路（过压/过流/反接）
      └── 存储（Flash + EEPROM 配置备份）

二、电调控制（RET Control）

2.1 RET 协议

RET 协议标准：3GPP TS 38.423 / O-RAN RET Profile
    
    RET 命令格式：
      下倾角：0° - 15°（步进 0.1°）
      方位角：0° - 360°（步进 0.1°）
      波束倾斜：±10°（步进 0.1°）
    
    通信接口：
      RS485（主流）
      Ethernet（新型 AAU）
      CAN Bus（部分厂商）

2.2 RET 精度要求

3GPP 规定：
      角度精度：±0.5°
      角度分辨率：0.1°
      角度重复性：±0.2°
      响应时间：< 200ms
    
    实际产品：
      精度：±0.3°
      分辨率：0.05°
      响应时间：< 100ms

2.3 电调下倾 vs 机械下倾

特性电调下倾（RET）机械下倾
调节范围0-15°固定 2-15°
调节精度0.1°固定档位
远程调节✅❌
对方向图影响小（窄化主瓣）大（畸变主瓣）
成本高低
可靠性中（有电机）高（无运动件）

三、波束控制

3.1 波束赋形参数

RCU 管理的波束参数：
      1. 波束指向角（θ, φ）
      2. 波束宽度（HPBW）
      3. 波束数量（多波束并发）
      4. 波束权重（幅度/相位）
    
    5G AAU 典型配置：
      64T64R → 最多 8 个独立波束
      每个波束可独立控制指向角和宽度

3.2 波束管理

波束切换流程：
      1. UE 上报 CSI（Channel State Information）
      2. gNB 计算最优波束权重
      3. RCU 下发权重到 T/R 模块
      4. T/R 模块更新移相器/衰减器
      5. 波束切换完成（< 1ms）

四、故障诊断与告警

4.1 监测参数

RCU 实时监测：
      温度：-40°C to +70°C（PA 温度、MCU 温度）
      电压：48V DC ±20%
      电流：0-50A（发射时）
      功率：当前发射功率
      VSWR：各通道驻波比
      PIM：无源互调水平
      风扇转速：（如果有）

4.2 告警级别

级别告警类型示例
Critical紧急PA 过热、电源故障
Major主要VSWR 过高、通道失效
Minor次要PIM 超标、温度偏高
Warning预警风扇转速异常

4.3 故障诊断流程

1. RCU 监测到异常
    2. 记录故障码和日志
    3. 通过 SNMP Trap / NETCONF 上报
    4. 网管系统告警
    5. 维护人员远程诊断或派单

五、通信接口

5.1 管理接口

接口协议速率说明
RS485RET Profile 2.0115.2kbps电调控制
EthernetSNMPv3 / NETCONF100Mbps网管通信
IPMIIPMI 2.0100kbps基站管理

5.2 前传接口（与 DU 通信）

AAU 与 DU 之间：
      CPRI（传统）：2.5Gbps/10Gbps
      eCPRI（5G 主流）：25Gbps
      
    数据内容：
      IQ 数据（波束赋形后的基带样本）
      控制信令（RET 命令、校准参数）
      状态上报（温度、功率、告警）

七、一句话总结

RCU 是基站天线的"大脑"——控制电调下倾、波束赋形、通道校准、故障诊断。5G AAU 中 RCU 功能已集成到 AAU 主控板中，不再独立。记住三个关键：① RET 精度 ±0.5° 是 3GPP 硬性要求 ② 波束切换 < 1ms 是用户体验底线 ③ 告警上报是运维的第一道防线。
